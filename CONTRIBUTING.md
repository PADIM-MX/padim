# Cómo Contribuir a PADIM

## 💡 ¿Quieres ayudar? Aquí tienes cómo

PADIM es un proyecto comunitario. Toda contribución es bienvenida, sin importar 
tu nivel de experiencia.

## 🎯 Formas de contribuir

| Tipo | Descripción | Experiencia requerida |
|------|-------------|:---------------------:|
| 🐛 Reportar bugs | Encuentra un problema y [abre un Issue](https://github.com/PADIM-MX/padim/issues) | Ninguna |
| 📝 Mejorar documentación | Corrige typos, clarifica ejemplos, traduce | Básica |
| 🧪 Escribir tests | Aumenta cobertura de tests | Media |
| 🔧 Reparar conectores | Cuando un portal cambia su estructura | Media |
| 🚀 Nuevas features | Trust Engine, CLI, validadores | Avanzada |

## 📋 Proceso

1. **Fork** el repositorio
2. **Crea una rama** con nombre descriptivo: `fix/ghost-detector-precision`, `feat/new-portal-connector`
3. **Haz tus cambios** — asegúrate de seguir las guías abajo
4. **Abre un Pull Request** contra la rama `main`
5. **Espera revisión** — un mantenedor revisará tu PR

## 📐 Guías de estilo

### Código Python
- Sigue PEP 8
- Type hints en todas las funciones públicas
- Docstrings en español (público objetivo MX)
- Nombres de variables en inglés

### Commits
- Prefijo descriptivo: `fix:`, `feat:`, `docs:`, `test:`, `refactor:`
- Primer línea < 72 caracteres
- Cuerpo opcional explicando el "por qué"

## 📜 CLA (Contributor License Agreement)

Al enviar un Pull Request, aceptas que tu contribución se publique bajo la 
licencia correspondiente del directorio donde se encuentre:
- `spec/` → CC0 1.0 Universal (dominio público)
- `engine/` → AGPL v3
- `cli/` → MIT
- Datos → ODbL v1.0

## ❓ Dudas

Abre un Issue o contacta a la comunidad. No hay preguntas tontas.

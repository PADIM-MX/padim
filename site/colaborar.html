Cómo colaborar con PADIM

PADIM es un protocolo abierto. No tiene dueño. Se construye con quien decide sumarse. Aquí tienes 4 formas de contribuir.

🔌 Conectores

Añadir un scraper

Los scrapers permiten extraer datos de portales inmobiliarios. Se distribuyen como plugins descargables desde un repositorio independiente. El core de PADIM nunca contiene código de scraping.

Requisitos del scraper

  • Python 3.11+ — el scraper debe ser compatible con Python 3.11 o superior
  • Función run() o main() — punto de entrada para el CLI
  • Dependencias explícitas — listarlas en un comentario al inicio del archivo
  • Licencia BSL 1.1 — misma licencia del repositorio de scrapers

Pasos

  1. Escribe tu scraper. Crea un archivo .py con la estructura esperada. Puedes usar los scrapers existentes como referencia.
  2. Verifica seguridad localmente. 
    padim scrapers verify mi_scraper.py
    El sistema escanea patrones peligrosos (os.system, eval, etc.), valida la estructura y genera el SHA256 para el manifiesto.
  3. Haz un Pull Request. El repositorio de scrapers está en Codeberg:

    codeberg.org/CamaradaMexicano389/mx-property-scrapers →

    Incluye tu archivo .py y el SHA256 generado.
  4. Espera revisión. Un mantenedor verificará que el SHA256 coincida, agregará la entrada al manifest.json y hará merge. Tu scraper estará disponible para todos los usuarios via padim scrapers install.

⚠️ Aviso legal: Los scrapers interactúan con sitios web de terceros. Como autor, eres responsable del uso de tu código y de cumplir los términos de servicio de los sitios a los que accedas. PADIM solo distribuye conectores; no controla su funcionamiento ni su legalidad.

🧠 Datos

Corregir propiedades

Los datos de PADIM se construyen con contribuciones de la comunidad. Si encuentras una propiedad con información incorrecta, puedes reportarla y ayudar a mejorar la calidad del acervo.

🔍

Explora en Quasar

Usa Quasar para buscar propiedades con trust score bajo. Las propiedades dudosas están marcadas para revisión.

✅

Verifica los datos

Compara precio, ubicación, fotos y descripción contra la fuente original. Anota las discrepancias.

📝

Reporta la corrección

Abre un issue en el repositorio principal con los datos correctos y la evidencia de la corrección.

Qué reportar

  • Precio incorrecto — el precio publicado no coincide con el real
  • Ghost listing — la propiedad ya no está disponible pero sigue listada
  • Ubicación errónea — la dirección o coordenadas no corresponden
  • Fotos incorrectas — las fotos no son de la propiedad anunciada
  • Datos faltantes — metros cuadrados, recámaras, baños sin especificar
📖 Estándar

Mejorar el estándar

El schema de datos, la especificación REST y el Trust Engine son el corazón de PADIM. Cualquier persona puede proponer cambios.

  • El schema está en spec/ y es CC0 dominio público. No requiere permiso para usarlo, modificarlo o redistribuirlo.
  • Las propuestas de cambio se discuten en GitHub Discussions. Cualquier cambio al schema requiere consenso de la comunidad.
  • El Trust Engine está en engine/ bajo AGPL v3. Si lo mejoras, contribuye tus cambios de vuelta a la comunidad.
  • El CLI y las herramientas están en cli/ bajo MIT. Máxima libertad para usarlo y adaptarlo.

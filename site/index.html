Tres niveles, un protocolo

01 — LO QUE ES

El Estándar

El schema de datos. La especificación REST. El formato de intercambio. Inmutable. Público. Gratis. No requiere mantenimiento. Existe y existirá.

02 — LO QUE HACE

Las Herramientas

Un CLI que se instala con pip install padim. Scrapea tu colonia. Valida la calidad. Contribuye al repositorio público. Evoluciona con la comunidad.

03 — LO QUE CONSTRUYE

Los Datos

El repositorio público de propiedades de México. Cada propiedad pasa por verificación de calidad. Los datos que no cumplen no aparecen. Los que cumplen son de todos.

⚡ Quick Start

pip install padim

# Scrapea propiedades de una colonia
padim scrape vivanuncios --colonia "Del Valle" --output delvalle.json

# Valida la calidad de los datos
padim validate delvalle.json

# Calcula trust score (0.0 - 1.0)
padim quality delvalle.json

# Sirve una API local con los datos
padim serve --port 8080

El CLI corre todo localmente en tu máquina. No enviamos datos a ningún servidor central sin tu permiso explícito.

🧠 Trust Engine

Cada propiedad contribuida pasa por un sistema de verificación de calidad antes de ser aceptada en el repositorio público.

Ghost

Propiedades que ya no existen (>90 días sin cambios). Impacto: -1.0

Fraude

Keywords sospechosas + precio anómalo. Impacto: -0.9

Precio

Precios irrealmente bajos o altos vs mercado. Impacto: -1.0

Agente

Agentes que dejan anuncios viejos para generar llamadas. Impacto: -0.6

Sin base de datos. Sin servidor central. Lógica pura que corre en tu máquina.

🤝 ¿Cómo participas?

1

Usa el estándar

Adopta el schema PADIM en tu proyecto, startup o universidad. No necesitas contribuir datos. Solo usar el estándar.

2

Contribuye datos

padim scrape desde tu computadora. Los datos se validan automáticamente. Si pasan calidad, se integran al dataset público.

3

Mantén un conector

Adopta un conector de scraping. Si se rompe, lo arreglas. Si nadie lo mantiene por 30 días, se marca como obsoleto.

📜 El Manifiesto

El mercado inmobiliario mexicano padece una fragmentación que no es accidental. Treinta plataformas compiten por el mismo tráfico. Ninguna comparte datos. Los portales mueren con propiedades que ya no existen.

Este desorden no es un fallo del sistema. Es el síntoma de un sistema diseñado para la escasez artificial de información.

PADIM no pide permiso. Publica el estándar. Construye las herramientas. Y deja que la comunidad haga el resto.
